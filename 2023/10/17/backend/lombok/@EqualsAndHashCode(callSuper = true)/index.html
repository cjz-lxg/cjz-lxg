@EqualsAndHashCode(callSuper = true)

发表于2023-10-17|更新于2023-11-09|java
|字数总计:341|阅读时长:1分钟|阅读量:

前言

写代码的时候因为我父子类都给标注了个@Data,然后idea提示我是否添加这个,以前就见过,不过没记录过,这次就顺手记录下了,

用法

The @EqualsAndHashCode annotation in Lombok is used to automatically generate implementations of the equals(Object other) and hashCode() methods inherited from Object. When you have a class hierarchy where a subclass extends a superclass, and both classes are annotated with @Data (which includes @EqualsAndHashCode), you may want to ensure that the equals and hashCode methods in the subclass take the fields of the superclass into account. This is where the callSuper attribute of the @EqualsAndHashCode annotation comes into play.

Here’s what happens:

  1. Without @EqualsAndHashCode(callSuper = true):

    • If you don’t include the @EqualsAndHashCode(callSuper = true) annotation on the subclass, the generated equals and hashCode methods will only consider the fields defined in the subclass itself, ignoring the fields defined in the superclass. This might not be the desired behavior if you want equality comparisons to consider the entire state of the object, including the state inherited from the superclass.

  2. With @EqualsAndHashCode(callSuper = true):

    • If you include the @EqualsAndHashCode(callSuper = true) annotation on the subclass, the generated equals and hashCode methods will also consider the fields defined in the superclass when performing equality comparisons and generating hash codes. This is typically the desired behavior when you have a class hierarchy and you want equality comparisons to be meaningful across the hierarchy.
