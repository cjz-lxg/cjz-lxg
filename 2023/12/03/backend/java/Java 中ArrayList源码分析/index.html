展开代码, 可以发现如下源码, 因为直接一口气解释, 可能让人看的头晕, 所以我这里选择截取每个代码片段来解释,同时, 并不介绍在指定位置插入元素, 因为实际上差不多, 看完以后也差不多会分析了, 无非是像判断是否扩容, 然后位移数组元素而已(其实主要原因是我写的时候漏了, 然后不想补了,毕竟确实简单)

可以看到, 这里的结构很简单, 其中, modCount代表的是这个ArrayList结构的改变, 根据注释写的, 它是用于检测在使用迭代器时是否发生并发异常的,因为这不是本篇文章的介绍重点, 所以不过多着墨了

s是当前ArrayList的元素数量, 当数组当前元素数量与数组的长度相等时, 触发grow()方法扩容,否则的话, 就是为在数组的末尾为它赋值

通过下面这段代码,我们可以看出扩容的逻辑, 如果当前的数组长度是0并且是默认的空数组, 那么新建一个数组, 该数组的大小取决于当前元素的数量+1和默认容量之间的比较, 而如果当前的数组长度大于0 或者 不是默认的空数组, 就会通过ArraysSupport.newLength()函数来得到扩容的大小, 这里使用ArraysSupport.newLength()是为了避免数组扩容时达到了JVM可以处理的上限, 所以你可以看到,假如说int prefLength = oldLength + Math.max(minGrowth, prefGrowth); // might overflow中的preLength的大小大于了SOFT_MAX_ARRAY_LENGTH, 就会尝试只扩容一个元素, 就是原来的数组长度加1, 而如果小于了的话 , 就会将数组的大小扩展到原来的两倍,而在扩容完毕后, 就会调用Arrays.copyOf来复制旧数组元素到新数组中去,而耗时的地方就在这, 所以我们应该尽量避免扩容.
