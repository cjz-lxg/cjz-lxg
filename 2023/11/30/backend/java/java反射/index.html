前言

今天简单复习下反射

什么是动态语言(✳)

动态语言就是程序在运行过程中可以改变其结构,具体表现为可以引进新的函数, 删去旧有的函数, 代表语言有python , JavaScript, 这种机制给语言带来立刻极大的灵活性, 具体表现为可以根据外部环境的变化来调整自身, 而C,C++都不是动态语言,而是静态语言,也就是编译时, 程序的行为就已经非常清晰了, 不受外部环境的影响, 这么看java应该算是半动态语言, 因为它提供了反射的机制, 使得程序可以根据环境的变化自行做出一定的调整

反射是什么

反射是一种机制, 指运行中的程序可以获取程序中的任意对象的具体的类,方法, 成员信息, 使运行中的程序做到在编译时无法做到的事

为什么需要反射

一个我自己亲自动手写过的rpc框架为例, 在当时的项目中, 我需要使用反射获取被指定注解的类并对他们进行实例化的操作, 这需要我使用反射获取类具体的注解信息, 同时, 有时候我需要根据客户端发来的方法和对应参数来进行反射调用具体方法, 而如果想要完成这些, 不使用反射就会显得很困难

所以我们可以粗略的看到一个结论, 反射往往使用在一个程序不知道外部需要它做什么时使用, 更深入的说, 也就是我们将程序看成一个状态机, 这个状态机会接受外部环境给他的参数, 并根据这些参数来完成对应的要求, 当完成要求以后, 会将信息返回给客户端,而我们平常写的程序它不需要外部参数, 也不接受外部参数. 所以我们可以看到, 平常的项目中, 我们很少见使用反射, 但是类似于spring这样的框架却会大量使用发射, 就在于spring本身开始运行时, 并不知道自己要运行什么, 只有当跑起来以后, 由外部程序告诉spring它需要做什么, 它才能知道自己需要做什么

反射的使用步骤

  • 获取想要操作的Class对象
  • 调用Class对象中的方法,获取诸如Method等类
  • 调用上一步获取的方法, 完成需求

反射涉及的类信息

  • Class 记录了类所有的信息, 其他如Filed, Method,Constructor,都需要从Class中获取
  • Field 记录了类的成员信息
  • Method 记录了类的方法信息
  • Constructor 记录了类的构造器信息
