节点流与处理流

发表于2023-11-09|更新于2023-11-09
|字数总计:315|阅读时长:1分钟|阅读量:

节点流 (Node Stream)

  • Direct Access: Node streams are low-level streams that directly interact with the specific I/O source or destination (like a file, memory, or socket).
  • Foundational: They form the foundation upon which other streams (processing streams) are built. They are typically used to open a communication path to the desired resource.
  • Examples: FileInputStream, FileOutputStream, FileReader, and FileWriter are all examples of node streams. They directly work with files on the filesystem.

处理流 (Processing Stream or Wrapper Stream)

  • Enhancement: Processing streams, also known as wrapper streams, are built on top of node streams and provide additional functionality like buffering, filtering, and transformation.
  • Decoration: They follow the Decorator Pattern, meaning they “wrap” a node stream to extend its capabilities without changing its interface.
  • Flexibility: They can be chained together to enhance I/O operations with multiple features. For example, a BufferedReader can wrap a FileReader to add buffering capability, which makes reading more efficient.
  • Examples: BufferedInputStream, BufferedOutputStream, BufferedReader, and BufferedWriter are all processing streams.

Here’s a brief summary of the Java I/O classification mentioned:

  • Flow Direction: Streams are divided into input streams for reading and output streams for writing.
  • Operation Unit: Streams are categorized based on whether they handle data in bytes (InputStream/OutputStream) or characters (Reader/Writer).
  • Role in I/O: Streams are further classified by their role as node streams (directly work with the data source) or processing streams (add functionality to another stream).

The Java I/O system is designed with a hierarchy that originates from four abstract classes:

  • InputStream/Reader: These are the base classes for all input streams, with the former handling byte streams and the latter handling character streams.
  • OutputStream/Writer: These are the base classes for all output streams, again with the former for byte streams and the latter for character streams.
