java中的异常分类及处理

发表于2023-11-29|更新于2023-11-29|java
|字数总计:738|阅读时长:2分钟|阅读量:

前言

准备面试, 开始复习下java的基础

什么是异常

简单来说, 异常就是程序出现错误, 无法继续向下运行, 此时程序会产生异常, 返回给用户错误信息

异常的分类

Throwable是异常的超类, 平常也没咋见用

Error

这个错误不由用户引起, 是因为java本身的问题所造成的, 当该异常产生时 , java虚拟机只能尽量保证程序能够平稳终止

Exception

这个错误分为两种, 一种时RuntimeException , 表示程序运行时错误, 另一种是CheckedException, 表示可能出现的错误

RuntimeException

这个异常典型大概有指针异常, 算术异常等,特征就是程序员写代码时犯的错误

CheckedException (✳)

这个异常我不知道为什么他要加个checked标签来着, 因为我感觉这个异常出现的时候往往不是程序本身出错, 而是外部环境出错, 这里是借用蒋炎岩的一种视角, 单独的一个程序就是一个状态机, 而当这个程序需要访问外部资源时 , 就是一个系统调用, 其含义就是状态机将cpu的控制权交给操作系统, 操作系统来帮助程序完成想要的操作, 可以看看这类典型的这类错误就是SqlException,IOException,可以发现, 这类错误基本都不是由程序本身引起的, 基本都是程序在访问外部资源时出错

异常的处理方式

这个分为两类处理, 一种是try catch ,也就是直接在本函数中直接处理, 另一种就是throws,将可能产生的异常丢给更上层的函数来处理

try catch

没啥好讲的,就是将可能抛出异常的函数放入try块, 然后用catch捕捉该异常, 然后做出对应的处理

throws

这个也没啥好讲的, 就是放在方法签名的后面, 可以跟不止一个异常, 表示可能出现的异常

附录

throw和throws(✳)

throw基本放在函数体里,当程序运行到它时, 将不会继续执行下面的代码, 而是选择直接抛出异常对象
throws基本放在函数签名后方, 表示可能出现的异常类

抛出异常有几种方式

共三种

  • 程序自抛出, 当你写了1/0时, 这类异常基本都是程序自动抛出的
  • 使用throw来抛出, 这类抛出的使用基本都是抛出一个自定义的异常对象
  • 使用throws来抛出, 和前两者不同, 前两者基本都是执行到异常所在行后发起的异常, 而throws是用来捕获当前方法体中出现的异常, 也就是throws本身不生产异常, 它只是前两种异常的搬运工
